This week I learned alot about module bundlers for JavaScript. In languages like Java and
Python you can use an import or include statements to inherit code from other files. JavaScript
did not come with this functionality because of security reasons. So instead, each time you wanted
to inherit a file you would need to load each one and share all variables on the global level. This is
obviously a lot of extra work so programmers started the project called CommonJS which was created to allow
modules in JavaScript and make inheritance easier. The most common version of CommonJS is node.js which is
still used today!


If you look at my first blog you can see that to include the moment class originally, you needed several
lines of code. However, with node.js you can load the file in with just one line!

var moment = require('moment');

If you simply include the above line in your .js file you can then use the moment methods freely! So node.js
gives JavaScript module usability which greatly reduces the effort needed to include other files. There is one
issue with this though and that is if you try to write this line in the browser you will get an error
so this line really only works in node.js. That is why we need module bundlers. A module bundler further
simplifies the process by building all the files and then giving you output that CAN be run in the browser!
In the commonly used module bundler "Webpack" you can run the following command:

$ ./node_modules/.bin/webpack index.js --mode=development

This command will look through your file and find all the "require" statements and build them together into
a single file. Now you can specify the output file in your browser and everything will work as planned!!
Even though this is mainly a quality of life addition, a module bundler was considered revolutionary for it's
time. When you need to use many different JavaScript methods a module bundler can save you loads of time and allow
for easier updates when you change your code!
